# i18n Rule — Always translate new text

This project uses a custom i18n system located in `src/lib/i18n/`.

## Rule

**Whenever you add any user-facing text to the project, you MUST also add the corresponding translation keys to all three locale files:**

- `src/lib/i18n/pt.json` — Portuguese (default)
- `src/lib/i18n/es.json` — Spanish
- `src/lib/i18n/en.json` — English

## How it works

- The `I18nProvider` in `src/lib/i18n/index.tsx` wraps the entire app and detects the user's browser language automatically via `navigator.language`.
- Language mapping: `es-*` → Spanish, `en-*` → English, everything else → Portuguese (default).
- Use the `useI18n()` hook to get the `t()` function in any component.
- Keys are dot-notated and match the JSON structure (e.g. `t("modal.title")`, `t("hero.badge")`).

## Adding new text — checklist

1. Add the key and Portuguese string to `pt.json`
2. Add the same key with Spanish translation to `es.json`
3. Add the same key with English translation to `en.json`
4. Use `t("your.key")` in the component — never hardcode strings

## Example

```json
// pt.json
{ "cta": { "new_button": "Saiba mais" } }

// es.json
{ "cta": { "new_button": "Saber más" } }

// en.json
{ "cta": { "new_button": "Learn more" } }
```

```tsx
const { t } = useI18n();
<button>{t("cta.new_button")}</button>
```
